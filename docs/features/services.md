# Dịch vụ hỗ trợ (Backing Services)

## Tổng quan

My PaaS cho phép tạo và quản lý các backing services phổ biến dưới dạng Docker container/service được quản lý tự động.

## Các loại service hỗ trợ

| Type | Image mặc định | Port | Mô tả |
|---|---|---|---|
| `postgres` | `postgres:16-alpine` | 5432 | PostgreSQL database |
| `redis` | `redis:7-alpine` | 6379 | In-memory cache/queue |
| `mysql` | `mysql:8` | 3306 | MySQL database |
| `mongo` | `mongo:7` | 27017 | MongoDB document store |
| `minio` | `minio/minio:latest` | 9000 | S3-compatible object storage |

## Lifecycle

```bash
# Tạo service
POST /api/services
{ "name": "my-db", "type": "postgres" }

# Khởi động
POST /api/services/:id/start

# Dừng
POST /api/services/:id/stop

# Xoá
DELETE /api/services/:id
```

Khi start, hệ thống:
1. Pull image (nếu chưa có)
2. Tạo container/Swarm service với volume persistent
3. Gắn vào `mypaas-network` để các app có thể kết nối
4. Tự tạo password ngẫu nhiên và cấp environment variables

## Service Linking

Liên kết service với project để tự inject connection strings:

```bash
POST /api/services/:serviceId/link/:projectId
{ "env_prefix": "DATABASE" }
```

Hệ thống tự inject các biến:

```
DATABASE_URL=postgres://user:pass@mypaas-svc-my-db:5432/mypaas
DATABASE_HOST=mypaas-svc-my-db
DATABASE_PORT=5432
DATABASE_USER=mypaas
DATABASE_PASSWORD=<auto-generated>
```

## Backup & Restore

```bash
# Tạo backup
POST /api/backups
{ "type": "service", "service_id": "abc123" }

# Download
GET /api/backups/:id/download

# Restore
POST /api/backups/:id/restore
```

System backup sao lưu toàn bộ database (SQLite file hoặc pg_dump).

## Marketplace Templates

One-click deploy từ template:

```bash
GET /api/marketplace
# → [{ id, name, description, services, config }]

POST /api/marketplace/:id/deploy
```

Templates cung cấp sẵn stack phổ biến (ví dụ: WordPress + MySQL, Ghost + MySQL).
